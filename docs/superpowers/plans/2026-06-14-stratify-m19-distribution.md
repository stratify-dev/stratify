# Stratify M19 (Distribution: prebuilt binaries, Homebrew, Marketplace) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Stratify as prebuilt cross-platform binaries on GitHub Releases, installable via Homebrew and a one-line installer, with a fast GitHub Action that downloads those binaries, and publish the Action to the GitHub Marketplace.

**Architecture:** cargo-dist generates a release workflow that, on a `v*` tag, cross-compiles the `stratify` binary, attaches tarballs + checksums to a GitHub Release, publishes a Homebrew formula to `stratify-dev/homebrew-tap`, and emits a shell installer. The composite Action is reworked to download the prebuilt binary instead of compiling from source. The first release is `v0.1.0` (matches the current workspace version, no bump). Marketplace publish is a manual checkbox on the release page.

**Tech Stack:** Rust workspace, [cargo-dist](https://opensource.axo.dev/cargo-dist/) (the `dist` tool), GitHub Actions, Homebrew. `cargo` is on PATH via `source "$HOME/.cargo/env"`.

- [ ] **Step 2: Add the dist config to the workspace `Cargo.toml`**

```toml
# cargo-dist configuration

ci = "github"
installers = ["shell", "homebrew"]
tap = "stratify-dev/homebrew-tap"
targets = [
    "aarch64-apple-darwin",
    "x86_64-apple-darwin",
    "x86_64-unknown-linux-gnu",
    "aarch64-unknown-linux-gnu",
    "x86_64-pc-windows-msvc",
]
# install to a stable dir the Action can add to PATH
install-path = "CARGO_HOME"
pr-run-mode = "plan"

[profile.dist]
inherits = "release"
lto = "thin"
```

Set `cargo-dist-version` to the exact version from Step 1 (run `dist --version`). If the installed cargo-dist rejects any key (run `dist plan` to find out), adjust to its schema. `tap` + `homebrew` in `installers` is the documented way to publish a formula to an external tap.

## Task 3: Rework the Action to download the prebuilt binary

**Files:**
- Modify: `action.yml`

- [ ] **Step 1: Replace the install steps**

Replace the two install steps (`Install Rust toolchain` and `Install stratify`) with a single download step that fetches the latest release's shell installer, installs to a known dir, and adds it to `PATH`, with a source-build fallback. Keep the `Run quality gate` step. Use the installer filename recorded in Task 1 Step 4 (shown here as `stratify-cli-installer.sh`). The new `runs:` block:

```yaml
runs:
  using: composite
  steps:
    - name: Install stratify (prebuilt, with source fallback)
      shell: bash
      run: |
        set -euo pipefail
        INSTALLER="stratify-cli-installer.sh"
        URL="https://github.com/stratify-dev/stratify/releases/latest/download/${INSTALLER}"
        export CARGO_HOME="${CARGO_HOME:-$HOME/.cargo}"
        if curl --proto '=https' --tlsv1.2 -fLsS "$URL" -o /tmp/stratify-installer.sh; then
          sh /tmp/stratify-installer.sh
        else
          echo "Prebuilt installer not available; building from source."
          if ! command -v cargo >/dev/null 2>&1; then
            curl --proto '=https' --tlsv1.2 -fLsS https://sh.rustup.rs | sh -s -- -y
          fi
          cargo install --git https://github.com/stratify-dev/stratify stratify-cli --locked
        fi
        echo "$CARGO_HOME/bin" >> "$GITHUB_PATH"

    - name: Run quality gate
      shell: bash
      run: |
        stratify check "${{ inputs.path }}" \
          --fail-on "${{ inputs.fail-on }}" \
          --format "${{ inputs.format }}"
```

The cargo-dist shell installer with `install-path = "CARGO_HOME"` installs `stratify` into `$CARGO_HOME/bin`, which the step appends to `GITHUB_PATH` so later steps find it. `releases/latest/download/<asset>` always redirects to the newest release's asset, so the Action stays version-independent.

- [ ] **Step 2: Update the `format` input description to mention sarif**

In `action.yml`, change the `format` input description:

```markdown
# Releasing Stratify

Releases are automated by cargo-dist. A pushed `v*` tag builds cross-platform
binaries, attaches them to a GitHub Release with checksums, updates the
Homebrew tap, and publishes a shell installer.

## Cut a release

1. Make sure `version` in the root `Cargo.toml` `[workspace.package]` matches
   the tag you are about to push (for example `0.1.0` for tag `v0.1.0`).
2. Tag and push:

   ```sh
   git tag v0.1.0
   git push origin v0.1.0
   ```

3. Watch the `Release` workflow in GitHub Actions. When it finishes, the
   release at `releases/latest` has the binaries, the `brew` formula is in
   `stratify-dev/homebrew-tap`, and the installer is published.

## Verify

```sh
brew install stratify-dev/tap/stratify
stratify --version
```

## Publish the Action to the GitHub Marketplace (once)

1. Open the published release page for the tag.
2. Ensure 2FA is enabled on your account and accept the GitHub Marketplace
   Developer Agreement when prompted.
3. Check "Publish this Action to the GitHub Marketplace".
4. Pick categories (Code quality, Continuous integration) and publish.

The Action's `name` ("Stratify Quality Gate") must be unique across the
Marketplace. If it is taken, change `name:` in `action.yml`, push, and re-tag.
```
